## 💡 使用技巧

### 1. 关键词匹配
使用准确的关键词可以更精确地匹配专家：
- ❌ "帮我写代码" (太模糊)
- ✅ "帮我用React写一个登录组件" (明确)

### 2. 提供上下文
提供更多上下文信息有助于专家给出更好的建议：
- ❌ "优化性能"
- ✅ "优化这个React列表组件的渲染性能，有1000条数据"

### 3. 逐步细化
从大问题逐步细化到具体问题：
1. "开发一个电商网站"
2. "设计商品列表页面"
3. "优化商品列表的无限滚动"

### 4. 参考推荐
系统会推荐相关专家，可以参考多个专家的建议。

### 5. 学习反馈
系统会从成功的解决方案中学习，使用越多越准确。

## 🔧 高级配置

### 启用/禁用进化系统

编辑 `~/.mimo/config.toml`:
```toml
[evolution]
enabled = true
autoLearn = true
autoUpdateReadme = true
```

### 自定义专家

在 `src/evolution/experts.ts` 中添加自定义专家。

### 清理学习数据

```bash
# 删除.mimo目录重置
rm -rf .mimo/

# 或在代码中调用清理方法
await selfLearning.cleanup(90); // 清理90天前的数据
```

## 🐛 故障排除

### 问题: 专家未匹配
**解决**: 
- 检查关键词是否准确
- 尝试使用不同的表述
- 查看 `/experts` 确认专家存在

### 问题: 匹配到错误的专家
**解决**:
- 使用更具体的关键词
- 提供更多上下文信息
- 参考系统推荐的其他专家

### 问题: 学习数据过多
**解决**:
- 定期清理旧数据
- 调整学习频率
- 使用 `cleanup()` 方法

### 问题: 性能影响
**解决**:
- 进化系统在后台运行，不影响使用
- 减少学习频率
- 清理 `.mimo/` 目录
